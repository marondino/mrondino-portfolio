Wildlife Studios needed a faster, more scalable way for game developers to create and configure games. Ahead of an internal Hackathon, stakeholders wanted a one-click solution — so I led research and design to map the game creation process end-to-end and deliver a usable CLI tool.

The Problem

Low Feature Awareness

We identified 31 distinct game features available to developers — but most weren't aware they existed. The main challenge: closing that visibility gap at scale.

Slow Setup & Cluster Management

Creating and configuring games took too long. Monitoring cluster health and visualizing updates lacked proper tooling. Developers needed faster feedback loops and simpler workflows.

Fragmented Documentation

Each feature had a different owner, and documentation lived in silos. Accessing information required navigating multiple channels — creating friction and slowing teams down.

Process

Mapping the Developer Journey

I ran workshops and research sessions to map the full game developer journey, identifying the most relevant features per game type and connecting patterns across all 31 services.

Wiki + Newsletter

To tackle feature invisibility, I designed a centralized Wiki covering all game features and their documentation. A recurring newsletter drove adoption — bringing developers to actively explore the available tooling.

CLI Design Documentation

I created a comprehensive CLI design guide defining behavioral patterns, voice and tone, and color-coded outputs per command type. This gave developers and designers a shared language for CLI interaction, aligned with Apple and Google style guides.

Metrics Definition

Defined the key metrics to track post-launch: CLI installation rate, bug metrics, conversion and retention rates, active users, churn, loading time, and security vulnerabilities.

Key Insight

There is no one-size-fits-all game creation flow. The CLI had to be modular and flexible enough to adapt to different game types — a fundamental shift from the initial assumption of a standardized process.
